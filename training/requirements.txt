# ---------------------------------------------------------------------------
# training/requirements.txt
#
# Dependencies for running the LeRobot / Pi0 / SmolVLA training pipeline as a
# REMOTE SageMaker Training Job (not the local `uv --project ../lerobot` flow).
#
# SageMaker behaviour: when `source_dir` (= this `training/` dir) contains a
# `requirements.txt`, the PyTorch DLC runs `pip install -r requirements.txt`
# INSIDE the container before invoking the entry point. So everything the
# training code imports at runtime must be resolvable from here.
#
# Target container: AWS PyTorch 2.1 DLC, py310, CUDA (g5.2xlarge = A10G, sm_86 →
# standard CUDA wheels work; the cu128/Blackwell special-case is ONLY for the
# on-prem RTX 5090 and does NOT apply here).
#
# NOTE: the DLC already ships a CUDA-enabled torch/torchvision. We intentionally
# do NOT pin torch here so pip keeps the container's GPU build. `lerobot` may try
# to pull its own torch; if you hit a torch downgrade-to-CPU during the build,
# add `--no-deps` handling or pin torch to the DLC version (see launch script
# notes). Verify with the `--self-test`/smoke run before a long job.
# ---------------------------------------------------------------------------
